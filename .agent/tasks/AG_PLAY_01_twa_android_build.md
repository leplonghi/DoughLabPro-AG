# AG_PLAY_01 — TWA Android Build (Google Play Store)

## Goal
Generate a signed Android App Bundle (AAB) for DoughLabPro using **Trusted Web Activity (TWA)** via Google's Bubblewrap CLI, extract the keystore SHA-256 fingerprint, update `assetlinks.json`, and produce all artifacts needed for Play Store submission.

---

## Context

DoughLabPro is a **PWA** (Progressive Web App) deployed to Firebase Hosting at:
- Production URL: `https://doughlabpro-fire.web.app`
- Package ID: `com.doughlabpro.app`
- `twa-manifest.json` is pre-configured in the project root

---

## Prerequisites (must be installed on the build machine)

```bash
# Java 17+
java -version  # must show 17 or higher

# Node.js 18+
node -v

# Android SDK (via Android Studio or command-line tools)
# ANDROID_HOME must be set, e.g.:
# export ANDROID_HOME=$HOME/Library/Android/sdk   # macOS
# export ANDROID_HOME=$HOME/Android/Sdk           # Linux
# export PATH=$ANDROID_HOME/tools:$ANDROID_HOME/platform-tools:$PATH

# Accept Android SDK licenses
$ANDROID_HOME/cmdline-tools/latest/bin/sdkmanager --licenses
```

---

## Step 1 — Install Bubblewrap CLI

```bash
npm install -g @bubblewrap/cli
bubblewrap --version  # confirm install
```

---

## Step 2 — Generate the Android Project

Run from the project root (where `twa-manifest.json` lives):

```bash
bubblewrap init --manifest https://doughlabpro-fire.web.app/manifest.webmanifest
```

When prompted:
- **Application Id**: `com.doughlabpro.app`
- **App name**: `DoughLabPro`
- **Launcher short name**: `DoughLabPro`
- **Host**: `doughlabpro-fire.web.app`
- **Start URL**: `/?utm_source=pwa`
- **Display mode**: `standalone`
- **Orientation**: `portrait`
- **Theme color**: `#65a765`
- **Background color**: `#ffffff`
- **Min SDK version**: `24`
- **Target SDK version**: `34`

This generates an `android/` folder with the full Android project.

---

## Step 3 — Generate Signing Keystore

```bash
# Run from project root
bubblewrap build
```

When prompted for signing key:
- **Keystore file path**: `./android.keystore`
- **Keystore alias**: `doughlabpro`
- **Key password**: Choose a secure password (16+ chars) — SAVE THIS
- **Keystore password**: Choose a secure password (16+ chars) — SAVE THIS
- **Name**: DoughLabPro
- **Organization**: DoughLabPro
- **Organizational Unit**: Mobile
- **Country**: BR
- **Validity**: 10000 days

---

## Step 4 — Extract SHA-256 Fingerprint

After build completes, extract the fingerprint:

```bash
keytool -list -v \
  -keystore ./android.keystore \
  -alias doughlabpro \
  -storepass YOUR_KEYSTORE_PASSWORD \
  | grep "SHA256"
```

The output will look like:
```
SHA256: AB:CD:EF:12:34:56:78:90:AB:CD:EF:12:34:56:78:90:AB:CD:EF:12:34:56:78:90:AB:CD:EF:12:34:56:78
```

---

## Step 5 — Update assetlinks.json

Replace the placeholder in `public/.well-known/assetlinks.json`:

```json
[
  {
    "relation": ["delegate_permission/common.handle_all_urls"],
    "target": {
      "namespace": "android_app",
      "package_name": "com.doughlabpro.app",
      "sha256_cert_fingerprints": [
        "AB:CD:EF:12:34:56:78:90:..."  // ← replace with actual SHA256
      ]
    }
  }
]
```

---

## Step 6 — Rebuild and Deploy

```bash
# Rebuild the web app with updated assetlinks.json
npm run build

# Deploy to Firebase Hosting
firebase deploy --only hosting

# Verify assetlinks.json is accessible:
curl https://doughlabpro-fire.web.app/.well-known/assetlinks.json
```

---

## Step 7 — Validate TWA Configuration

Validate the Digital Asset Links:

```bash
# Open in browser (replace with actual fingerprint):
# https://digitalassetlinks.googleapis.com/v1/statements:list?source.web.site=https://doughlabpro-fire.web.app&relation=delegate_permission/common.handle_all_urls
```

Should return a JSON response with `matched` statements.

---

## Step 8 — Build the AAB for Play Store

```bash
bubblewrap build
```

Output file: `app-release-bundle.aab`

This is the file you upload to Google Play Console.

---

## Step 9 — Verify the Build

```bash
# Check AAB was generated:
ls -lh app-release-bundle.aab

# Optionally test APK on device:
bubblewrap install
```

---

## Notes

- The `twa-manifest.json` in the project root has all settings pre-configured
- **NEVER commit `android.keystore` to git** — add it to `.gitignore`
- **SAVE THE KEYSTORE PASSWORDS** — losing them means you cannot update the app on Play Store
- If using **Play App Signing** (recommended): upload the AAB to Play Console, and Google manages key rotation
- After first upload, the app package name `com.doughlabpro.app` cannot be changed
- If the production URL is a custom domain instead of `doughlabpro-fire.web.app`, update `host` in `twa-manifest.json` and `assetlinks.json` accordingly
